# 美食社交小程序 - 部署文档

## 1. 系统要求

### 1.1 硬件要求
- **最低配置**：2核CPU，4GB内存，50GB存储
- **推荐配置**：4核CPU，8GB内存，100GB存储
- **生产环境**：8核CPU，16GB内存，500GB存储

### 1.2 软件要求
**后端服务器**：
- Node.js >= 18.0.0 （建议使用 20.x LTS）
- MongoDB >= 5.0 （用于数据持久化）
- Redis >= 6.0 （用于缓存和会话管理）
- Python >= 3.11 （AI服务，当前为stub）
- node-cron （定时任务依赖）
- Winston （日志系统）

**开发环境**：
- 微信开发者工具
- Git
- VS Code或类似编辑器

## 2. 环境配置

### 2.1 开发环境配置

#### 克隆项目

## 3. 后端部署

### 3.1 安装依赖
```bash
cd backend
npm install
```

### 3.2 数据库初始化
确保MongoDB和Redis服务已启动：

```bash
# 启动MongoDB（示例）
mongod --dbpath ./data/db

# 启动Redis（示例）
redis-server
```

### 3.3 启动后端服务

**开发模式**：
```bash
npm run dev
```

**生产模式**：
```bash
# 构建应用
npm run build

# 启动服务
npm start
```

**使用PM2管理进程**：
```bash
# 安装PM2
npm install -g pm2

# 启动应用
pm2 start ecosystem.config.js
```

创建 `ecosystem.config.js`：
```javascript
module.exports = {
  apps: [{
    name: 'foodie-social-backend',
    script: 'src/app.js',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
};
```

## 4. AI服务部署

### 4.1 安装Python依赖
```bash
cd ai-service
pip install -r requirements.txt
```

### 4.2 模型下载和配置
```bash
# 创建必要的目录
mkdir models
mkdir output
mkdir cache

# 下载预训练模型（示例）
python download_models.py
```

### 4.3 启动AI服务
**开发模式**：
```bash
python main.py
```

**生产模式**：
```bash
# 使用Gunicorn（推荐）
gunicorn -k uvicorn.workers.UvicornWorker main:app -b 0.0.0.0:8000

# 或直接使用Uvicorn
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 4.4 使用Docker部署
创建 `Dockerfile`：
```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

构建和运行：
```bash
docker build -t foodie-social-ai .
docker run -p 8000:8000 foodie-social-ai
```

## 6. 生产环境部署

### 6.1 使用Docker Compose
创建 `docker-compose.yml`：
```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:5.0
    container_name: foodie-social-mongo
    restart: always
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: password
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db

  redis:
    image: redis:7-alpine
    container_name: foodie-social-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: foodie-social-backend
    restart: always
    ports:
      - "3000:3000"
    environment:
      - MONGODB_URI=mongodb://mongodb:27017/foodie_social
      - REDIS_URI=redis://redis:6379
      - NODE_ENV=production
    depends_on:
      - mongodb
      - redis
    volumes:
      - ./backend/logs:/app/logs

  ai-service:
    build:
      context: ./ai-service
      dockerfile: Dockerfile
    container_name: foodie-social-ai
    restart: always
    ports:
      - "8000:8000"
    volumes:
      - ./ai-service/models:/app/models
      - ./ai-service/output:/app/output
      - ./ai-service/cache:/app/cache

  nginx:
    image: nginx:alpine
    container_name: foodie-social-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./ssl:/etc/nginx/ssl
    depends_on:
      - backend
      - ai-service

volumes:
  mongodb_data:
  redis_data:
```

### 6.2 Nginx配置
创建 `nginx.conf`：
```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log;
access_log /var/log/nginx/access.log;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    # 后端API代理
    upstream backend {
        server backend:3000;
    }

    # AI服务代理
    upstream ai_service {
        server ai-service:8000;
    }

    server {
        listen 80;
        server_name your-domain.com;
        
        # HTTP重定向到HTTPS
        return 301 https://$server_name$request_uri;
    }

    server {
        listen 443 ssl http2;
        server_name your-domain.com;
        
        # SSL配置
        ssl_certificate /etc/nginx/ssl/cert.pem;
        ssl_certificate_key /etc/nginx/ssl/key.pem;
        ssl_protocols TLSv1.2 TLSv1.3;
        ssl_ciphers HIGH:!aNULL:!MD5;
        
        # 后端API
        location /api/v1 {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # WebSocket支持
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
        
        # AI服务
        location /ai {
            proxy_pass http://ai_service;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}
```

### 6.3 启动生产环境
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 8. 故障排除

### 8.1 常见问题

**Q: MongoDB连接失败**
A: 检查MongoDB服务状态和连接字符串。

**Q: WebSocket连接失败**
A: 检查Nginx配置和SSL证书。

**Q: AI服务响应慢**
A: 增加AI服务worker数量，优化模型。

**Q: 小程序API调用失败**
A: 检查服务器域名配置和SSL证书。

### 8.2 日志查看
```bash
# 后端日志
tail -f logs/combined.log

# AI服务日志
tail -f ai-service/logs/service.log

# Nginx访问日志
tail -f /var/log/nginx/access.log

# Docker日志
docker-compose logs -f backend
docker-compose logs -f ai-service
```
